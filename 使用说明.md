# 使用说明

## 1. 启动
```bash
uv run python main.py
```

浏览器访问：
- 本机：`http://127.0.0.1:18765`
- 局域网：`http://<工控机IP>:18765`

## 2. 用例管理
- 在“导入测试用例”卡片选择并加载用例。
- 点击“创建用例”创建一个默认空用例，并自动进入编辑模式。
- 点击“删除用例”可删除当前选择用例（测试运行中不可删除）。

## 3. 编辑规则
- 用例加载后默认只读。
- 点击“编辑用例”进入编辑模式。
- 点击“保存用例”后自动回到只读模式。
- 仅只读模式允许开始测试。
- 只读模式会隐藏新增/删除/上下移按钮。

## 4. 编辑结构
- Item 与 Step 都是可折叠卡片。
- 默认全部折叠；保存后保留当前折叠/展开状态。
- 名称在卡片顶栏直接编辑。
- `Step.type`（Action/Verify）在顶栏编辑。
- `Step.plugin` 与 `Step.action` 为下拉选择：
  - 先选插件，再选动作。
  - 切换插件后，动作会自动校正为该插件可用动作。
- 参数区：
  - `params` 使用 JSON 文本。
  - `params` 无需填写 `__htf_context`，框架会在运行时自动注入。
  - `expr` 仅 Verify 步启用。

## 5. 新增与排序
- 编辑模式下，使用卡片式 `+` 添加 Item 或 Step。
- 顶栏 `↑/↓` 可调整顺序。
- 可删除 Item/Step。

## 6. 执行与结果
- 点击“开始测试”执行。
- 点击“开始测试”后会先清空当前用例卡片上的上一轮 Item/Step 结果状态。
- 点击“停止测试”后，当前 Step 结束后停止。
- 运行状态：
  - `Pass`、`Fail`、`Error`、`Stopped`
- 结果在左侧 Item/Step 右侧实时显示。

## 7. 历史与导出
- “执行与历史”显示历史摘要。
- 点击历史中的测试用例名称（超链接）打开详情页。
- 在详情页执行导出：
  - JSON：`/api/reports/{run_id}.json`
  - Excel：`/api/reports/{run_id}.xlsx`

## 8. 插件配置与自检
- “插件”区域按插件卡片展示，点击卡片头可折叠/展开。
- 点击“编辑配置”后才可编辑该插件配置 JSON。
- 点击“保存配置”后，配置会写入并自动重新加载到该插件。
- 可对单个插件点击“自检”，也可使用“全部自检”批量自检已加载插件。
- 卡片右侧显示自检状态：
  - `未自检` / `通过` / `失败` / `异常` / `不支持`
- 插件卡片不显示自检返回详情，仅显示自检状态。

## 9. 示波器截图归档
- 使用插件 `oscilloscope` 的 `capture_screenshot` 工步可保存截图。
- 截图文件默认保存到 `reports/<run_id>/artifacts/`。
- 插件返回结构建议使用 `result.artifacts` 数组。
- 历史详情页会按附件类型渲染：`image` / `log` / `csv` / `waveform` / `metric`。
- `waveform` 可用于展示电压/电流等时序数据（按通道显示采样点数量）。
- `capture_mode=placeholder` 用于联调；接入真实仪器时使用 `capture_mode=cli` 并配置 `cli_command`。
- 截图插件通过运行时上下文 `params.__htf_context` 获取 `run_id` 等信息（由框架自动注入）。
