# 插件开发文档

本文面向在 Mochi HTF 中开发设备插件的工程师，覆盖：
- 插件入口与生命周期
- `run/action/params` 约定
- 配置与自检
- 运行时上下文 `params.__htf_context`
- 附件结果 `result.artifacts`
- 常见坑与调试方式

## 1. 放置位置与发现规则

- 插件目录：`./plugins`
- 发现方式：框架刷新插件时扫描 `plugins/*.py`（以下划线开头文件会忽略）
- 插件模块需满足其一：
  - `create_plugin() -> object`
  - `class Plugin`（可实例化）

建议文件命名与插件名一致，例如：
- 文件：`plugins/power_ctrl.py`
- 插件名：`power_ctrl`

## 2. 最小可用插件

```python
from __future__ import annotations

from typing import Any


class Plugin:
    name = "hello_demo"

    def actions(self) -> list[str]:
        return ["ping"]

    def run(self, action: str, params: dict[str, Any]) -> dict[str, Any]:
        if action == "ping":
            return {"ok": True, "message": "pong"}
        raise ValueError(f"Unsupported action: {action}")


def create_plugin() -> Plugin:
    return Plugin()
```

## 3. 推荐接口（完整）

插件可选实现以下方法：
- `name: str`：插件名（建议必填）
- `actions() -> list[str]`：动作列表（建议实现，供 UI 下拉）
- `run(action: str, params: dict) -> Any`：执行动作（必需）
- `default_config() -> dict`：默认配置
- `config_schema() -> dict`：配置项描述（用于 UI 显示）
- `set_config(config: dict)`：接收当前生效配置
- `self_check(...) -> bool | dict`：插件自检

> 说明：框架会先合并 `default_config + 已保存配置`，再调用 `set_config`（如果实现了该方法）。

## 4. `run` 函数签名兼容

框架兼容以下 `run` 签名：
- `run(action, params)`
- `run(action, params, plugin_config=...)`
- `run(action, params, config=...)`
- `run(action, params, <第三个位置参数>)`

但推荐你使用：
- `set_config(...)` 接收配置
- `run(action, params)` 保持简洁

## 5. 配置与自检约定

### 5.1 `default_config`

必须返回 JSON 对象（`dict`），例如：

```python
def default_config(self) -> dict:
    return {"address": "", "timeout": 5.0}
```

### 5.2 `config_schema`

建议返回：

```python
def config_schema(self) -> dict:
    return {
        "fields": [
            {"key": "address", "type": "string", "label": "设备地址", "description": "VISA 或 IP"},
            {"key": "timeout", "type": "number", "label": "超时(秒)", "description": "命令超时"},
        ]
    }
```

### 5.3 `self_check`

可返回：
- `True/False`
- `{"ok": bool, ...}`

推荐返回结构化信息：

```python
def self_check(self) -> dict:
    return {"ok": True, "message": "device ready"}
```

## 6. 参数约定与保留字段

每个 Step 的 `params` 都是插件动作参数。

框架执行时会自动注入：
- `params.__htf_context`（保留字段）

当前字段：
- `run_id`
- `case_id`
- `item_id`
- `step_id`
- `step_name`
- `reports_dir`

用途：
- 生成与本次执行绑定的文件名
- 把附件落到 `reports/<run_id>/artifacts/`

建议：
- 不要在业务参数里使用 `__htf_` 前缀
- 对未知字段保持忽略，不要报错

## 7. 返回值与附件规范（重点）

普通返回值可为任意 JSON 可序列化对象。

若要让报告页展示附件，推荐返回：

```json
{
  "ok": true,
  "artifacts": []
}
```

### 7.1 `artifacts` 通用字段

每个附件对象建议包含：
- `type`：附件类型
- `label`：展示名
- `path`：相对路径（推荐 `reports/<run_id>/artifacts/...`）
- `url`：可访问链接（推荐 `/api/reports/{run_id}/artifacts/{artifact_path}`）

### 7.2 支持类型

当前报告页支持：
- `image`：显示图片预览
- `log`：显示文本日志
- `csv`：显示 CSV 文本
- `waveform`：显示波形通道/采样点统计
- `metric`：显示标量指标（名称/值/单位）

### 7.3 示例

`metric`（单值）：

```json
{
  "type": "metric",
  "name": "voltage",
  "value": 13.5,
  "unit": "V"
}
```

`metric`（多通道）：

```json
{
  "type": "metric",
  "label": "multi_channel_voltage",
  "items": [
    {"name": "CH1_voltage", "value": 13.5, "unit": "V"},
    {"name": "CH2_voltage", "value": 13.3, "unit": "V"}
  ]
}
```

`waveform`：

```json
{
  "type": "waveform",
  "label": "v_i_waveform",
  "series": [
    {"name": "voltage", "unit": "V", "points": [{"t_ms": 0, "value": 12.1}]},
    {"name": "current", "unit": "A", "points": [{"t_ms": 0, "value": 1.2}]}
  ]
}
```

## 8. 典型插件模板（含配置/自检/附件）

```python
from __future__ import annotations

from pathlib import Path
from typing import Any


class Plugin:
    name = "my_device"

    def __init__(self) -> None:
        self._config = self.default_config()

    def default_config(self) -> dict[str, Any]:
        return {"address": ""}

    def config_schema(self) -> dict[str, Any]:
        return {
            "fields": [
                {"key": "address", "type": "string", "label": "设备地址", "description": "IP/VISA"}
            ]
        }

    def set_config(self, config: dict[str, Any]) -> None:
        merged = self.default_config()
        merged.update(config or {})
        self._config = merged

    def actions(self) -> list[str]:
        return ["read_voltage"]

    def self_check(self) -> dict[str, Any]:
        return {"ok": True, "message": "ready"}

    def run(self, action: str, params: dict[str, Any]) -> dict[str, Any]:
        if action != "read_voltage":
            raise ValueError(f"Unsupported action: {action}")

        value = 13.5
        return {
            "ok": True,
            "artifacts": [
                {"type": "metric", "name": "voltage", "value": value, "unit": "V"}
            ],
        }


def create_plugin() -> Plugin:
    return Plugin()
```

## 9. 错误处理建议

- 参数校验失败：`raise ValueError("...")`
- 设备通信失败：`raise RuntimeError("...")`
- 非预期异常：让异常抛出即可，框架会记录为 Step `Error`

执行超时由框架控制（Step `timeout`），超时会中止插件子进程并记为 `Error`。

## 10. 调试与联调建议

- 先写最小 action（返回固定值）验证插件加载
- 再接真实设备命令
- 自检先验证“通信可达”，不要做危险动作
- 需要存文件时优先走 `reports/<run_id>/artifacts/` 归档

## 11. 验收清单

- 插件能在 UI “插件”区被发现
- `actions` 能出现在 Step 动作下拉
- `self_check` 可正常返回
- Step 执行成功/失败可被正确记录
- 附件（如图片/指标）可在历史详情页显示
- Excel 导出“附件”列可看到路径或链接
